Builder Design Pattern

Definition

The Builder Pattern is a creational design pattern that allows the step-by-step construction of complex objects. It separates the construction of a complex object from its representation, enabling the same construction process to create different representations.

Usage

  • When an object needs to be created with many optional parameters.
  • When you want to avoid constructors with too many parameters.
  • When building complex objects step by step.

Example in C#

public class Car
{
    public string Engine { get; set; }
    public int Wheels { get; set; }
    public bool HasGPS { get; set; }
}

public class CarBuilder
{
    private Car _car = new Car();

    public CarBuilder SetEngine(string engine) { _car.Engine = engine; return this; }
    public CarBuilder SetWheels(int wheels) { _car.Wheels = wheels; return this; }
    public CarBuilder AddGPS() { _car.HasGPS = true; return this; }
    public Car Build() { return _car; }
}

var car = new CarBuilder()
    .SetEngine("V8")
    .SetWheels(4)
    .AddGPS()
    .Build();

Advantages

  • Improves code readability with step-by-step construction.
  • Makes the creation of complex objects easier.
  • Avoids large constructors with many parameters.

Disadvantages

  • Can make the code more complex compared to simple constructors.
  • Requires creating a separate Builder class.
